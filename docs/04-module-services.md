# 6 — Module: Services (`app/services/`)

The **services** module contains all business logic. Each service owns one aggregate and:

1. Validates business rules (not just types — that's schemas' job)
2. Enforces multi-tenancy (`org_id` filter on every query)
3. Coordinates DB + Redis + LLM + query engine
4. Returns ORM models or DTOs

**No HTTP knowledge here.** Services don't know about FastAPI, requests, or responses.

## File map

```
app/services/
├── auth_service.py            # Signup, login, refresh, logout
├── user_service.py            # Profile, role updates, password change
├── invitation_service.py      # Invite tokens, acceptance flow
├── connection_service.py      # DB connection CRUD + encryption
├── session_service.py         # GenAI session lifecycle
├── query_service.py           # Orchestrates NL query execution
├── report_service.py          # Saved queries
├── dashboard_service.py       # Dashboard CRUD + items
└── websocket_manager.py       # In-process WS connection registry
```

## `auth_service.py`

### `signup(db, payload) -> AuthResponse`

1. Check email not already used (within new org)
2. Create `Organization` (plan=free)
3. Create `User` (role=admin, status=active)
4. Set `org.owner_id`
5. Hash password with bcrypt
6. Issue JWT pair → return tokens + user + org

### `login(db, email, password) -> TokenPair`

1. Find user by email (case-insensitive)
2. Verify bcrypt
3. Check `user.status == active`
4. Issue token pair

### `refresh(refresh_token) -> TokenPair`

1. Decode token, verify type == "refresh"
2. Check Redis for jti blacklist
3. Blacklist old jti (replay protection)
4. Issue new pair

### `logout(refresh_token)`

Adds the refresh token's jti to Redis blacklist with TTL = remaining lifetime.

## `invitation_service.py`

### `invite(db, ctx, email, role) -> str` (invite token)

1. Admin only
2. Check email not already in org
3. Create `User(status=pending)` row
4. Create JWT invite token (24h expiry)
5. (App calls `utils.email.send_invite()` separately)

### `accept_invite(db, token, password) -> AuthResponse`

1. Decode invite token
2. Find user by id + email
3. Hash password, set status=active
4. Issue access + refresh

## `connection_service.py`

### `create(db, ctx, payload) -> DBConnection`

1. Encrypt username + password with Fernet
2. Insert `db_connections` row (org_id from ctx)
3. Optionally test connection right away

### `test_connection(db, ctx, conn_id) -> TestConnectionResponse`

1. Load connection (must be in ctx.org_id)
2. Decrypt creds
3. Try to open + close a trial connection (timeout 5s)
4. Update `last_tested_at`
5. Return success/error + latency

### `grant_access(db, ctx, conn_id, target_user_id)`

Inserts `db_connection_access` row. `user_id=null` means org-wide.

### `update(db, ctx, conn_id, payload)` / `delete(...)`

Standard CRUD. Always filter by `org_id`.

## `session_service.py`

### `create(db, ctx, connection_id, title) -> GISession`

1. Verify user has access to connection (via `db_connection_access` or org-wide)
2. Insert session with empty context_window
3. (Title can be auto-generated by `llm.title_generator` on first message)

### `append_to_context(db, session, role, content)`

Appends `{role, content}` dict to `context_window` JSONB. Trims oldest entries if total tokens exceed `SESSION_CONTEXT_WINDOW_SIZE`.

### `list_sessions(db, ctx)` / `get_history(db, ctx, session_id)`

Filtered by `user_id` (sessions are per-user, not per-org by default).

## `query_service.py` ⭐ Most important service

### `run_query(db, ctx, session_id, prompt, on_event=None) -> RunQueryResponse`

This is the **core orchestration**. Called by both the REST endpoint and the WebSocket handler. The optional `on_event` callback emits status updates for streaming.

#### Steps:

```python
async def run_query(db, ctx, session_id, prompt, on_event=None):
    # 1. Load session + check access
    session = await get_session(db, ctx, session_id)
    
    # 2. Notify
    await on_event(StatusMsg(stage="extracting_intent"))
    
    # 3. LLM intent extraction
    intent = await llm.intent_extractor.extract(
        prompt, context=session.context_window
    )
    
    # 4. Load template
    template = template_loader.get(intent.template_id)
    
    # 5. Bind parameters (validates types, no string SQL)
    bound_sql, params = parameter_binder.bind(template, intent.params)
    await on_event(SqlMsg(sql=bound_sql))
    
    # 6. Execute on target DB (streaming)
    await on_event(StatusMsg(stage="executing"))
    rows = []
    async for batch in executor.stream(connection, bound_sql, params):
        rows.extend(batch)
        await on_event(DataMsg(rows=batch, is_last_batch=False))
    
    # 7. Generate insight
    await on_event(StatusMsg(stage="generating_insight"))
    insight = await llm.insight_generator.generate(rows, intent)
    await on_event(InsightMsg(text=insight))
    
    # 8. Persist QueryHistory
    await persist_history(db, session, prompt, intent, bound_sql, status="success")
    
    # 9. Update session context
    await session_service.append_to_context(db, session, "user", prompt)
    await session_service.append_to_context(db, session, "assistant", insight)
    
    return RunQueryResponse(...)
```

#### Error handling

Each step is wrapped to ensure history is always persisted (even on failure) with `execution_status` = `error` / `rate_limited`.

## `report_service.py`

A **report** is a saved query template + parameters + column display config.

| Method | Notes |
|--------|-------|
| `create(db, ctx, payload)` | Validates template exists; sets `created_by` |
| `run(db, ctx, report_id, override_params)` | Re-uses query engine without LLM step |
| `add_column(db, ctx, report_id, column)` | Position auto-incremented |
| `update(...)` / `delete(...)` | Standard CRUD |
| `set_schedule(db, ctx, report_id, data)` | Sets `refresh_interval_days`, recalculates `next_refresh_at` |
| `manual_refresh(db, ctx, report_id, conn_id)` | Runs query directly, saves snapshot, updates report's `next_refresh_at` |
| `list_snapshots(db, ctx, report_id, limit)` | Fetches list of query snapshots for a report (metadata only) |
| `get_snapshot_detail(db, ctx, report_id, snapshot_id)` | Fetches single snapshot with complete row results (`raw_data`) |
| `run_due_reports(db_session)` | Background worker job (invoked by APScheduler hourly) that executes outstanding scheduled reports |

Reports do **not** invoke the LLM — they're parameterized templates.
For a complete overview of background scheduled report processing and system-level db connection bypass mechanics, see [Data Execution & Gateway Agent Flows](./09-data-execution-and-gateway-flows.md).

## `websocket_manager.py`

### `WebSocketManager`

In-process registry of active WebSocket connections, keyed by `session_id`.

```python
manager = get_ws_manager()

await manager.register(session_id, ws)
await manager.send(session_id, msg)
await manager.unregister(session_id, ws)
```

### Why in-process?

- Single-instance deployments (most users): zero overhead
- Multi-instance: swap implementation to use Redis pub/sub. Interface stays the same.

### Lifecycle

- `init_ws_manager()` called on startup
- `shutdown_ws_manager()` called on graceful shutdown — closes all open WS with code 1001

## Conventions

| Rule | Why |
|------|-----|
| Every method takes `db: AsyncSession` and `ctx: TenantCtx` | Explicit dependencies, easy to test |
| Every query filters by `ctx.org_id` | Hard tenancy |
| RBAC checks at top of method | Fail fast |
| Raise `AppError` subclasses, never plain Exception | Clean HTTP mapping |
| Return ORM models OR domain DTOs (not Pydantic schemas) | Service layer is HTTP-agnostic |
| No `print()` — use `log = get_logger(__name__)` | Structured logs |
| No `@router` decorators here | Services are HTTP-free |

## Service example template

```python
# app/services/widget_service.py
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select
from app.api.v1.dependencies.tenancy import TenantCtx
from app.core.database.models import Widget
from app.core.exceptions import NotFound

async def list_widgets(db: AsyncSession, ctx: TenantCtx) -> list[Widget]:
    stmt = select(Widget).where(Widget.org_id == ctx.org_id).order_by(Widget.created_at.desc())
    return list((await db.execute(stmt)).scalars().all())

async def get_widget(db: AsyncSession, ctx: TenantCtx, widget_id: UUID) -> Widget:
    stmt = select(Widget).where(Widget.id == widget_id, Widget.org_id == ctx.org_id)
    widget = (await db.execute(stmt)).scalar_one_or_none()
    if widget is None:
        raise NotFound("Widget not found")
    return widget
```
